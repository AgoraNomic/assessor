Rules to the contrary\nnotwithstanding, a person CANNOT become Registered if eir basis overlaps\nwith that of any current player.\n\n}\n\n\n[\n\nThis fixes a bug where two group persons [A B] and [B C] could be both\nbe players, since neither is a part of the other.\n\nIn order to let this work, this also prohibits simultaneous\nregistrations. This is done now because simultaneous registrations would\nbreak the check for overlapping persons (if [A B] and [B C] could\nregister simultaneously, neither would overlap with any existing player,\neven if they overlapped with each other). As a more general matter,\nsimultaneous registrations seem likely to result in bugs, and\nregistration order has been used as a tiebreak before (e.g. for Spaaace).\n\n]"
