            "author": "kiako",
            "coauthors": [
            ],
            "text": "[No numerical switch uses the \"default to 0\" fallback, except for\nCrystal's identity, which is always set to a different value before it\ncan be used. Considering the \"default to null\" fallback, 5 switches\nexplicitly create and define a \"none\" instance, and two more define\nsimilar values under different names (\"unassigned\" and \"vacant\"). It is\nclear then that neither of these fallbacks take on the semantic meaning\nof \"a default value meant to be used\".\n\nIn this case, it would make more sense to unambiguously allow Crystal\nidentity to default to \"null\", assigning that fallback the semantic\nmeaning of \"This switch must have a default value, but it does not\nmatter what it is because it is never used in practice.\"\n\nJanet noted that this would break mathematical operations if a numerical\nswitch ends up using the default, but under this new semantic meaning I\nbelieve the outcome is what is expected: since \"null + 3\" is undefined,\nthe switch fails to be flipped, and remains \"null\". (Defaulting to 0, on\nthe other hand, would give the guise that the default should be\nconsidered a usable instance.)]\n\nAmend Rule 2509 (\"Agoran Numbers\") by removing the following paragraph:\n{\n     If 0 is in the specified values for a numerical switch and no\ndefault value is otherwise specified, 0 is the default value for that\nswitch.\n}"
